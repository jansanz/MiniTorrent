Detailed Description

Represent a Handshake message according to Bittorrent Protocol. It has the form: <pstrlen><pstr><reserved><info_hash><peer_id>

  • # pstrlen: string length of <pstr>, as a single raw byte -
  • # pstr: string identifier of the protocol -
  • # reserved: eight (8) reserved bytes -
  • # info_hash: 20-byte SHA1 hash of the info key in the metainfo file -
  • # peer_id: 20-byte string used as a unique ID for the client -
