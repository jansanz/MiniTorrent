synchronized void jBittorrentAPI.DownloadTask.keepAliveSent (  ) 

Fired when a keep-alive message has been sent by the MessageSender. If at the time the keep-alive was sent, this peer has not received any message from the remote peer since more that 3 minutes, the remote peer is considered as dead, and we close the connection, then inform the DownloadManager that this connection timed out...

Otherwise, inform the DownloadManager that this task is still alive and has not been used for a long time...
